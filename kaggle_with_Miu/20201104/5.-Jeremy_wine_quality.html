2. Model selection¶

First of all, it should be noted that the ultimate goal of this analysis is to precisely classify the good wine, which means the positive class is more important than the negative one. Hence, we could select one of these PR AUC, ROC AUC, or F-beta score as the main metric. However, the EDA in the first section has indicated that the data has imbalanced classes so we should not use ROC AUC because this metric only works well when the ratios of both classes are relatively similar and both classes detection is equally important. Furthermore, we would like to predict the probabilities rather than the label so that we could adjust the threshold to deal with the imbalanced class problem. Therefore, the PR AUC should be chosen to be the main metric because it fits to our goal and works well on the heavily imbalanced data.

In the first step of the analysis, we will try to fit the data on several models with default setting to find out which one returns the best Precision - Recall curve. One way to do that is to compute the Average Precision score of each model and compare the result. The Average Precision score is the avarage precision at all such possible thresholds of the PR curve, and it is an useful metric to compare how well models are ordering the predictions, without considering any specific decision threshold. Another important reason to compute Average Precision score rather than F1 score is that we might want to favour the precision. The final model should be precision-oriented because False Negative hurts performance more than False Positive. On other words, we want to reduce as much as possible the number of true good wines that are wrongly predicted as bad one because the wrong prediction might negatively affect the sales and reputation of good wine brewers.

Before applying a number of supervised learning methods, I would like to try 2 unsupervised learning models on the data as well. The first model is K-means clustering, and the second one is more sophisticated that it illustrates the data into 2 dimensional, which is called t-SNE.

Instead of calculating the average precision score for each model manually, the helper function defined below will make the job faster.

In this analysis, the Over-sampling technique and stratified k fold cross validation will be applied to reduce the effect of imbalanced data on the model performance. To avoid leakage, the Over-sampling will only be applied after doing stratified K-fold.

In [15]:
# Support function to compute average F1 score from stratified K-fold cross validation
def avg_precision_score(model, k):
  """Compute the average average precision score from the given model by using
  stratified K-fold cross validation
  Parameters: model: the model
              K: the number of folds
  """
  import numpy as np
  from sklearn.model_selection import StratifiedKFold
  from sklearn.metrics import average_precision_score
  from imblearn.over_sampling import SMOTE
  from sklearn.preprocessing import MinMaxScaler

  # Create stratified k-fold object
  kfold = StratifiedKFold(n_splits=k, shuffle=True, random_state=0)
  
  # Compute F1 scores from each fold
  avg_precision_scores = []
  for train_index, test_index in kfold.split(X, y):
    # Assign train & test dataset
    X_train, X_test = X.loc[train_index], X.loc[test_index]
    y_train, y_test = y.loc[train_index], y.loc[test_index]
    
    # Define oversample strategy
    oversample = SMOTE(sampling_strategy=0.5)
    # Fit and the transformation on the train data
    X_over, y_over = oversample.fit_resample(X_train, y_train)

    # Scale data by MinMaxScaler
    scaler = MinMaxScaler()
    X_over_train_scaled = scaler.fit_transform(X_over)
    X_over_test_scaled = scaler.transform(X_test)

    # Fit the model to the current fold data
    model.fit(X_over_train_scaled, y_over)
    # Use the trained model to predict the unseen dataset
    y_pred = model.predict(X_over_test_scaled)

    # Compute the F1 score of the current fold and add it to the list of F1 scores
    avg_precision = average_precision_score(y_test, y_pred)
    avg_precision_scores.append(avg_precision)
  
  # Compute the average F1 score
  result = np.mean(avg_precision_scores)
  
  return result

In general, the average precision score of three models are not substaintially different. Therefore, I personally favour the Support Vector Machine in this case even though it has returned the slightly lower average precision score than Random Forest Classifier. There are two main reasons for my model selection and the first one is that Support Vector Machine model could work well with Polynomial features and avoid overfitting easily by adjusting its regularization parameter (C). The second reason is that SMV returns result faster than RF.

In the final model below, after over-sampling the training data to the ratio of 0.5 (reduce the class imbalance from 1:6 to 1:2) and transforming the data into Polynomial features at the degree of 2, the grid search cross-validation will be applied on the Support Vector Machine in order to find the best parameters that return the highest average precision score. In the last step, the Precision-Recall curve will be computed and we will try to find the best threshold that favours the precision score at the acceptable level of recall.
